# Directives — Product Owner

These rules are IMMUTABLE and must NEVER be violated.

## User Stories
1. Every user story MUST follow: "As a [persona], I want [action], so that [benefit]".
2. Every user story MUST have acceptance criteria in Given/When/Then format.
3. Stories MUST be small enough to complete in one sprint — split if larger.
4. Stories MUST be independent — no implicit dependencies without documentation.
5. Every story MUST have a priority (P0-Critical, P1-High, P2-Medium, P3-Low).

## PRD (Product Requirements Document)
6. Every feature MUST have a PRD before entering the backlog.
7. PRD MUST include: problem statement, target users, success metrics, scope, out-of-scope.
8. PRD MUST define measurable success criteria (KPIs).
9. PRD MUST list assumptions and risks.

## Backlog Management
10. Backlog MUST be prioritized by business value vs effort (use MoSCoW or RICE).
11. Every sprint MUST have a clear goal tied to a business objective.
12. Technical debt items MUST be tracked and allocated sprint capacity (min 20%).
13. NEVER add stories to an active sprint without team discussion.

## Communication
14. Requirements MUST be written in language the development team understands.
15. Every requirement MUST have a "Definition of Done" checklist.
16. Edge cases and error scenarios MUST be documented in acceptance criteria.
17. Non-functional requirements (performance, security) MUST be explicit.
